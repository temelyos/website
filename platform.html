HOW TEMELYOS WORKS

Three pieces. One platform.

A shell application for power users with a reusable UI kit, a backend, and a module system that lets anyone extend the platform — including you.

01 — FRONTEND
Shell application
A power-user interface with a tabbed workspace. Each tab is a micro-app; the shell hosts them, routes navigation, and exposes an API so every micro-app can call the integrated AI assistant.
Tabbed workspace — switch between tasks without losing work; every tab is a live micro-app.
Power-user UI — tree navigation, multi-tab work, keyboard-friendly. Built for people running the business.
Integrated AI assistant — every micro-app can call it via API. Model-agnostic: OpenAI, Anthropic, or any LLM, switchable with zero code changes.
Reusable UI kit — the same components that power the shell, available to your custom external UI.

02 — CORE
Backend
Use the backend without the shell. Pair it with a custom external UI for end users while admins use the shell — one consistent product on one foundation.
Shell API — micro-apps interact with the shell, navigation, and the assistant through a stable API.
Service–provider system — capabilities are defined as services; modules register providers that implement them, so implementations can be swapped without touching the code that consumes them.
Multi-tenant by default — data isolation is enforced at the platform level, across every query and API call.
Security built in — users, roles, hierarchical permissions, and token-based auth at every layer, not bolted on.
03 — EXTENSIBILITY
Module system
Third parties add micro-apps and services without modifying the platform. Native modules are uploaded and run inside Temelyos; existing self-hosted apps can be mounted from anywhere, written in any language.
Micro-apps, grouped — each item in the workspace tree is a separate micro-app, registered by a module.
Self-hosted apps — mount existing apps written in any language, on any backend, hosted anywhere.
First-class integration — no distinction between built-in modules and the ones you build.
